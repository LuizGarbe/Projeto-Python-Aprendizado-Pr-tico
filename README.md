🗂️ Estrutura do Projeto
Com base na organização do diretório, o projeto está dividido em:

validar dados/: Implementações focadas em tratamento de entradas, garantindo que o programa receba dados no formato correto e trate erros de execução.

comoFuncionsListas/: Exercícios práticos sobre manipulação de listas, métodos de ordenação, fatiamento e iterações.

teste_de_OO/: Introdução à Orientação a Objetos (OO), explorando a criação de classes, métodos e a lógica de instanciar objetos.

testes/: Scripts dedicados a testar funcionalidades isoladas antes da implementação final.

python2/: Evolução de conceitos e scripts de transição/estudo de versões.

📜 Scripts Principais
Na raiz do projeto, encontram-se os arquivos de execução base:

inicial.py: Ponto de partida dos estudos.

principal.py: Script unificador das funcionalidades aprendidas.

palavras.txt: Base de dados simples utilizada para exercícios de leitura e extração de strings.

🚀 O que estou praticando
Validação de Dados: Uso de try/except e condicionais para evitar que o código quebre com inputs inválidos.

Manipulação de Arquivos: Extração de informações contidas em arquivos externos (como o palavras.txt).

Lógica de Programação: Estruturação de fluxos de decisão e repetição eficientes.

Fundamentos de OO: Organização do código em classes para torná-lo reutilizável e escalável.
